Detailed Description

Logging functionalities.

There are 5 different log levels:

  • ERROR: use for error messages that capture situations that could compromise the correct functioning of the program.
    e.g. "Failed to generate encryption key"
  • WARNING: use for messages that capture situations that require the user attention, but don't yet compromise the correct functioning of the program.
    e.g. "Lost network connection"
  • INFO: use for messages that contain important information for the user on the correct functioning of the program.
    e.g. "Enrolled to Quarklink"
  • DEBUG: use for messages that contain information only valuable to a programmer that is working on the code. You can also use this log level for temporary messages while working on specific parts of the code.
    e.g. "Received data: bbbbb"
  • VERBOSE: use for messages that highlight operation in smaller details than debug, also only useful for programmers.
    e.g. "Start initialisation of communications", "Connecting", "Connected"
Note
Logs follow the same structuring as Espressif's ESP32 logging library.

Implementations of logging functions This is a generic function that makes it easier to adapt the log to any platform.

However, it is not optimised to reduce memory consumption: A buffer containing the formatted output string is malloc'd here (outstr) and a pointer to that buffer is passed to the platform-specific function (ql_printf) that outputs it. Most likely, the platform-specific function will itself allocate more memory for the buffer that is being passed. To optimise, only use this cqLog_printf version if memory is not an issue, otherwise it is recommended to implement a native alternative of cqLog_printf. // TODO provide example / do it for some of the boards
