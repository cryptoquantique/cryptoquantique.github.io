Detailed Description

Collection of all the platform-specific functions.

When adding support for a new platform, the functions declared in this file need to be implemented.
Inside platform_specific/, define a new .c/.cpp file named after the platform containing the implementations.

◆ ql_printf()

int ql_printf ( const char *  str)

Print the argument string to the desired output.

Note
This function is needed for the log prints and should be as easy as calling printf("%s", str);.
However, some platforms do not support/implement the stdio printf and it needs to be replaced by an alternative function (e.g. Arduino's Serial.print(str)).
No newline needed at the end of the text.
Parameters
[in]strThe text to print, in the form of an already formatted c-string.
Returns
A negative number for error, 0 or higher for success (normally the number of char printed).

int ql_https_put ( const quarklink_context_t *  quarklink,
const char *  url,
const char *  jbody,
char *  rxBuffer 
)

Send an HTTPS PUT request to the specified URL.

The request can only have a JSON payload.
The function always specifies the following two headers: Content-Type: application/json Accept: application/json

Note
Use the details in the quarklink context to establish mTLS. In particular: quarklink->iotHubRootCert for the CA cert quarklink->deviceCert for the certificate quarklink_getDeviceKey() for the key
Parameters
[in]quarklinkThe quarklink context
[in]urlThe URL to send the request to
[in]jbodyThe body in JSON format
[out]rxBufferThe buffer that will contain the response
Return values
0for SUCCESS
-1for FAILURE

Send an HTTPS GET request to the specified URL.


If failed (response is not 200 or 301), the request is re-iterated a maximum number of times (maxRetries), after which it is considered failed.

Note
Use the details in the quarklink context to establish mTLS. In particular: quarklink->iotHubRootCert for the CA cert quarklink->deviceCert for the certificate quarklink_getDeviceKey() for the key
Parameters
[in]quarklinkThe quarklink context
[in]urlThe URL to send the request to
[in]maxRetriesThe maximum number of retries for the request
[out]rxBufferThe buffer that will contain the response
Return values
0for SUCCESS
-1for FAILURE
